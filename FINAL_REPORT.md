## Project Summary

EQ Switch is a lightweight Windows tray utility for EverQuest that lets you instantly flip between multiple game clients with a single keypress. It includes a full suite of multi-boxing tools: Picture-in-Picture overlays via DWM thumbnails, multi-monitor window management, process priority and CPU affinity control, a Video Settings editor for eqclient.ini, and configurable hotkeys for every action. Built with AutoHotkey v2 — single .exe, no installation, no system footprint.

## Key Features

- **Window switching** — configurable global hotkey scoped to EQ, cycles through all visible clients
- **Global Switch hotkey** — pulls EQ to front from any app, cycles clients on repeat
- **Multi-monitor window distribution** — one EQ window per monitor, toggle hotkey to spread/swap/stack
- **Picture-in-Picture** — live DWM thumbnail overlay of alt clients, click-through with Ctrl+drag repositioning
- **PiP border** — thin colored border for visibility, configurable color
- **Process Manager** — set priority (High/AboveNormal/etc) and CPU affinity per client
- **Video Settings editor** — resolution presets, title bar offset, WindowedMode toggle for eqclient.ini
- **Middle-click tray actions** — single/triple-click paradigm for Notes and PiP toggle
- **Launch N clients** — configurable client count with staggered launch and auto-fix
- **Help window** — cyberpunk-styled scrollable reference with keyboard shortcuts
- **Settings GUI** — compact layout with EQ path, hotkeys, PiP config, launch options, and more

## Lessons Learned

### AHK v2 Global Scoping
Nested functions need explicit `global` declarations or variables silently become local. This bit us with TARGET_MONITOR, g_pipCtrlWatch, and g_dblClickUpIgnore — all appeared to work in simple tests but failed at runtime when called from nested contexts.

### AHK Hotkey GUI Control Limitation
The built-in Hotkey GUI control cannot represent bare keys like `\` or `]` — `.Value` returns empty string. Fix: preserve the old binding when the control returns empty instead of clearing it.

### 64-bit Struct Offsets
APPBARDATA lParam is at offset 40 (not 36) due to HWND alignment padding in 64-bit AHK. Always verify struct layouts when porting Win32 code from 32-bit examples.

### WM_MBUTTONUP vs WM_MBUTTONDOWN on Win11
Win11's system tray does not reliably deliver WM_MBUTTONDOWN (0x207). Must use WM_MBUTTONUP (0x208) exclusively for middle-click detection. Original v1.7 used 0x208; v2.0 broke it by switching to 0x207.

### DWM Thumbnail PiP
DwmRegisterThumbnail provides zero-copy live window previews. Key patterns: swap sources in-place to avoid flicker, use WS_EX_TRANSPARENT for click-through with Ctrl-toggle for drag, wrap .Hwnd access in try for race conditions during destruction.

### eqclient.ini Quirks
WindowedModeYOffset only takes effect when toggling window mode (windowed to fullscreen and back). Must flip WindowedMode=FALSE then back to TRUE for offset changes to apply. X offset of 0 is stable; -8 shifts too far left.

### Config Migration
Ship a ReadKey() helper that wraps IniRead with defaults and try/catch. When renaming config sections, auto-migrate old keys on first load and delete the old section.
