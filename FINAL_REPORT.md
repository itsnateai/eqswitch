## Project Summary

EQSwitch is an EverQuest multiboxing window manager for the Shards of Dalaya emulator community. Originally written in AutoHotkey v2, it was ported to C# (.NET 8 WinForms) for better VirusTotal compatibility, type safety, and debuggability. It compiles to a single portable exe with no installer or runtime dependency.

## Key Features

- **Window switching** via global hotkeys (RegisterHotKey) and low-level keyboard hook (WH_KEYBOARD_LL)
- **Window arrangement** in grid layout (single monitor) or one-per-monitor (multi-monitor)
- **CPU affinity management** with P-core/E-core awareness (Intel hybrid CPU optimized)
- **PiP overlay** using DWM thumbnails (GPU-composited, zero CPU overhead)
- **Staggered client launching** with configurable delay and auto-arrange
- **8-tab dark-themed settings GUI** (General, Hotkeys, Layout, Affinity, Launch, PiP, Paths, Characters)
- **eqclient.ini video settings editor**
- **Character profiles** with export/import (JSON)
- **Process manager** with live affinity display and force-apply
- **Config migration** from AHK eqswitch.cfg on first run
- **Persistent file logging** with rotation
- **79 unit tests** covering core logic

## Lessons Learned

### P/Invoke Safety
- Always use `GetWindowLongPtr`/`SetWindowLongPtr` (not the 32-bit variants) — silent failures on x64.
- Always dispose Process objects from `GetProcessById()` / `GetProcessesByName()` — they wrap OS handles. The polling code runs every 500ms.
- COM objects need `Marshal.FinalReleaseComObject()` in finally blocks.
- Keyboard hook delegates must be stored as fields — GC collection silently kills the hook.

### Low-Level Keyboard Hooks
- Windows requires LL hook callbacks to return within ~300ms or it silently removes the hook permanently. Never do blocking work in the callback — post to UI thread via `SynchronizationContext.Post()`.
- Global key hooks swallow keys system-wide. Always add guards (e.g., "only when target app is running").

### DWM Thumbnails
- GPU-composited, zero CPU — far superior to screen capture for PiP overlays.
- `WS_EX_TRANSPARENT` makes a window permanently click-through at the OS level. For selective click-through, use `WM_NCHITTEST` overrides instead.

### Config Reload
- Stop and **dispose** old timers (not just Stop). Leaked timers accumulate.
- Full reload sequence: stop timers → unregister hotkeys → unhook → apply config → re-register → re-hook → start timers.

### EQ-Specific
- EQ resets its own CPU affinity after launch — need retry mechanism.
- `eqclient.ini` uses ANSI encoding — writing UTF-8 corrupts it.
- Window title changes from `"EverQuest"` to `"EverQuest - CharName"` on login — poll for name updates.
